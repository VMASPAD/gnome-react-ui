# Tabs

Un componente para alternar entre paneles relacionados. `Tabs.Indicator` se coloca **dentro** de `Tabs.List` y usa CSS variables (`--active-tab-left`, `--active-tab-width`, etc.) para posicionarse automáticamente sobre el tab activo.

<Tabs.Root className="overflow-hidden rounded-xl border border-gray-200 bg-white shadow-sm" defaultValue="preview">
  <Tabs.List className="relative z-0 flex border-b border-gray-200 bg-gray-50 px-2">
    <Tabs.Tab className="relative flex h-10 min-w-[96px] cursor-pointer items-center justify-center px-4 text-center text-sm font-medium whitespace-nowrap text-gray-500 outline-none select-none transition-colors duration-150 hover:text-gray-800 focus-visible:z-10 focus-visible:outline focus-visible:outline-2 focus-visible:outline-blue-600 focus-visible:rounded data-[active]:text-gray-900 data-[active]:font-semibold" value="preview">Preview</Tabs.Tab>
    <Tabs.Tab className="relative flex h-10 min-w-[96px] cursor-pointer items-center justify-center px-4 text-center text-sm font-medium whitespace-nowrap text-gray-500 outline-none select-none transition-colors duration-150 hover:text-gray-800 focus-visible:z-10 focus-visible:outline focus-visible:outline-2 focus-visible:outline-blue-600 focus-visible:rounded data-[active]:text-gray-900 data-[active]:font-semibold" value="code">Code</Tabs.Tab>
  </Tabs.List>
  <Tabs.Panel className="flex min-h-[180px] items-center justify-center p-6 outline-none" value="preview">

  <Tabs.Panel className="p-0 outline-none" value="code">
    ```tsx
    import { Tabs } from "gnome-ui/tabs";

    export function TabsUnderline() {
      return (
        <Tabs.Root defaultValue="overview">
          <Tabs.List className="relative flex border-b border-border">
            <Tabs.Tab value="overview" className="data-[active]:text-foreground">
              Overview
            </Tabs.Tab>
            <Tabs.Tab value="projects" className="data-[active]:text-foreground">
              Projects
            </Tabs.Tab>
            {/* Indicator DENTRO de List, usa CSS variables del componente */}
            <Tabs.Indicator
              className="absolute bottom-0 h-0.5 rounded-full bg-primary transition-[left,width] duration-200"
              style={{
                left: "var(--active-tab-left)",
                width: "var(--active-tab-width)",
              }}
            />
          </Tabs.List>
          <Tabs.Panel value="overview">...</Tabs.Panel>
          <Tabs.Panel value="projects">...</Tabs.Panel>
        </Tabs.Root>
      );
    }
    ```
  </Tabs.Panel>
</Tabs.Root>

# Anatomy

```tsx
import { Tabs } from "gnome-ui/tabs";

<Tabs.Root defaultValue="tab1">
  <Tabs.List>
    <Tabs.Tab value="tab1" />
    <Tabs.Tab value="tab2" />
    <Tabs.Indicator />  {/* dentro de List */}
  </Tabs.List>
  <Tabs.Panel value="tab1" />
  <Tabs.Panel value="tab2" />
</Tabs.Root>
```

    ```tsx

          <Tabs.List className="flex rounded-xl bg-muted p-1 gap-1">

              <Tabs.Tab
    ```

## TabsIndicator

A visual indicator that can be styled to match the position of the currently active tab.
Renders a `&lt;span&gt;` element.

Documentation: [Base UI Tabs](https://base-ui.com/react/components/tabs)

### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **className** | `string \| (...) => string)` | — | CSS class applied to the element, or a function that<br/>returns a class based on the component’s state. |
| **render** | `ReactElement> \| ComponentRenderFn` | — | Allows you to replace the component’s HTML element<br/>with a different tag, or compose it with another component.<br/><br/>Accepts a `ReactElement` or a function that returns the element to render. |
| **renderBeforeHydration** | `boolean` | `false` | Whether to render itself before React hydrates.<br/>This minimizes the time that the indicator isn’t visible after server-side rendering. |

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **className** | `string \| (...) => string)` | — | CSS class applied to the element, or a function that<br/>returns a class based on the component’s state. |

## TabsRoot

Groups the tabs and the corresponding panels.
Renders a `&lt;div&gt;` element.

Documentation: [Base UI Tabs](https://base-ui.com/react/components/tabs)

### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **className** | `string \| (...) => string)` | — | CSS class applied to the element, or a function that<br/>returns a class based on the component’s state. |
| **defaultValue** | `any` | `0` | The default value. Use when the component is not controlled.<br/>When the value is `null`, no Tab will be active. |
| **render** | `ReactElement> \| ComponentRenderFn` | — | Allows you to replace the component’s HTML element<br/>with a different tag, or compose it with another component.<br/><br/>Accepts a `ReactElement` or a function that returns the element to render. |
| **value** | `any` | — | The value of the currently active `Tab`. Use when the component is controlled.<br/>When the value is `null`, no Tab will be active. |
| **orientation** | `Orientation` | `'horizontal'` | The component orientation (layout flow direction). |
| **onValueChange** | `(...) => void` | — | Callback invoked when new value is being set. |

## TabsTab

An individual interactive tab button that toggles the corresponding panel.
Renders a `&lt;button&gt;` element.

Documentation: [Base UI Tabs](https://base-ui.com/react/components/tabs)

### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **className** | `string \| (...) => string)` | — | CSS class applied to the element, or a function that<br/>returns a class based on the component’s state. |
| **render** | `ReactElement> \| ComponentRenderFn` | — | Allows you to replace the component’s HTML element<br/>with a different tag, or compose it with another component.<br/><br/>Accepts a `ReactElement` or a function that returns the element to render. |
| **value*** | `any` | — | The value of the Tab. |
| **disabled** | `boolean` | — | Whether the Tab is disabled.<br/><br/>If a first Tab on a `&lt;Tabs.List&gt;` is disabled, it won't initially be selected.<br/>Instead, the next enabled Tab will be selected.<br/>However, it does not work like this during server-side rendering, as it is not known<br/>during pre-rendering which Tabs are disabled.<br/>To work around it, ensure that `defaultValue` or `value` on `&lt;Tabs.Root&gt;` is set to an enabled Tab's value. |
| **nativeButton** | `boolean` | `true` | Whether the component renders a native `&lt;button&gt;` element when replacing it<br/>via the `render` prop.<br/>Set to `false` if the rendered element is not a button (e.g. `&lt;div&gt;`). |
