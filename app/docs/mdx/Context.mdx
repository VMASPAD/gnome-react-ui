# Context

## ContextMenuRoot

A component that creates a context menu activated by right clicking or long pressing.
Doesn’t render its own HTML element.

Documentation: [Base UI Context Menu](https://base-ui.com/react/components/context-menu)

### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **onOpenChange** | `(...) => void` | — | Event handler called when the menu is opened or closed. |
| **children** | `ReactNode \| PayloadChildRenderFunction<unknown>` | — | The content of the popover.<br/>This can be a regular React node or a render function that receives the `payload` of the active trigger. |
| **open** | `boolean` | — | Whether the menu is currently open. |
| **disabled** | `boolean` | `false` | Whether the component should ignore user interaction. |
| **orientation** | `MenuRootOrientation` | `'vertical'` | The visual orientation of the menu.<br/>Controls whether roving focus uses up/down or left/right arrow keys. |
| **triggerId** | `string` | — | ID of the trigger that the popover is associated with.<br/>This is useful in conjunction with the `open` prop to create a controlled popover.<br/>There's no need to specify this prop when the popover is uncontrolled (i.e. when the `open` prop is not set). |
| **loopFocus** | `boolean` | `true` | Whether to loop keyboard focus back to the first item<br/>when the end of the list is reached while using the arrow keys. |
| **actionsRef** | `RefObject<MenuRootActions>` | — | A ref to imperative actions.<br/>- `unmount`: When specified, the menu will not be unmounted when closed.<br/>   Instead, the `unmount` function must be called to unmount the menu manually.<br/>  Useful when the menu's animation is controlled by an external library.<br/>- `close`: When specified, the menu can be closed imperatively. |
| **handle** | `MenuHandle<unknown>` | — | A handle to associate the menu with a trigger.<br/>If specified, allows external triggers to control the menu's open state. |
| **defaultOpen** | `boolean` | `false` | Whether the menu is initially open.<br/><br/>To render a controlled menu, use the `open` prop instead. |
| **onOpenChangeComplete** | `(...) => void` | — | Event handler called after any animations complete when the menu is closed. |
| **defaultTriggerId** | `string` | — | ID of the trigger that the popover is associated with.<br/>This is useful in conjunction with the `defaultOpen` prop to create an initially open popover. |
| **highlightItemOnHover** | `boolean` | `true` | Whether moving the pointer over items should highlight them.<br/>Disabling this prop allows CSS `:hover` to be differentiated from the `:focus` (`data-highlighted`) state. |
| **closeParentOnEsc** | `boolean` | `false` | When in a submenu, determines whether pressing the Escape key<br/>closes the entire menu, or only the current child menu. |

## ContextMenuTrigger

An area that opens the menu on right click or long press.
Renders a `&lt;div&gt;` element.

Documentation: [Base UI Context Menu](https://base-ui.com/react/components/context-menu)

### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **className** | `string \| (...) => string)` | — | CSS class applied to the element, or a function that<br/>returns a class based on the component’s state. |
| **render** | `ReactElement> \| ComponentRenderFn` | — | Allows you to replace the component’s HTML element<br/>with a different tag, or compose it with another component.<br/><br/>Accepts a `ReactElement` or a function that returns the element to render. |
