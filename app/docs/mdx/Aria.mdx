# Aria

## AriaCombobox



### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| **filter** | `(...) => boolean` | — | Filter function used to match items vs input query. |
| **id** | `string` | — | The id of the component. |
| **open** | `boolean` | — | Whether the popup is currently open. Use when controlled. |
| **disabled** | `boolean` | `false` | Whether the component should ignore user interaction. |
| **grid** | `boolean` | `false` | Whether list items are presented in a grid layout.<br/>When enabled, arrow keys navigate across rows and columns inferred from DOM rows. |
| **inline** | `boolean` | `false` | Whether the list is rendered inline without using the popup. |
| **onOpenChange** | `(...) => void` | — | Event handler called when the popup is opened or closed. |
| **name** | `string` | — | Identifies the field when a form is submitted. |
| **loopFocus** | `boolean` | `true` | Whether to loop keyboard focus back to the input when the end of the list is reached while using the arrow keys. The first item can then be reached by pressing &lt;kbd&gt;ArrowDown&lt;/kbd&gt; again from the input, or the last item can be reached by pressing &lt;kbd&gt;ArrowUp&lt;/kbd&gt; from the input.<br/>The input is always included in the focus loop per [ARIA Authoring Practices](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/).<br/>When disabled, focus does not move when on the last element and the user presses &lt;kbd&gt;ArrowDown&lt;/kbd&gt;, or when on the first element and the user presses &lt;kbd&gt;ArrowUp&lt;/kbd&gt;. |
| **modal** | `boolean` | `false` | Determines if the popup enters a modal state when open.<br/>- `true`: user interaction is limited to the popup: document page scroll is locked and pointer interactions on outside elements are disabled.<br/>- `false`: user interaction with the rest of the document is allowed. |
| **actionsRef** | `RefObject<Actions>` | — | A ref to imperative actions.<br/>- `unmount`: When specified, the combobox will not be unmounted when closed.<br/>Instead, the `unmount` function must be called to unmount the combobox manually.<br/>Useful when the combobox's animation is controlled by an external library. |
| **defaultOpen** | `boolean` | `false` | Whether the popup is initially open.<br/><br/>To render a controlled popup, use the `open` prop instead. |
| **onOpenChangeComplete** | `(...) => void` | — | Event handler called after any animations complete when the popup is opened or closed. |
| **selectionMode** | `SelectionMode` | `'none'` | How the combobox should remember the selected value.<br/>- `single`: Remembers the last selected value.<br/>- `multiple`: Remember all selected values.<br/>- `none`: Do not remember the selected value. |
| **selectedValue** | `ComboboxItemValueType<Value, Mode>` | — | The selected value of the combobox. Use when controlled. |
| **defaultSelectedValue** | `ComboboxItemValueType<Value, Mode>` | — | The uncontrolled selected value of the combobox when it's initially rendered.<br/><br/>To render a controlled combobox, use the `selectedValue` prop instead. |
| **onSelectedValueChange** | `(...) => void` | — | Callback fired when the selected value of the combobox changes. |
| **fillInputOnItemPress** | `boolean` | — | INTERNAL: When `selectionMode` is `none`, controls whether selecting an item fills the input. |
| **itemToStringValue** | `(...) => string` | — | When the item values are objects (`&lt;Combobox.Item value=&#123;object&#125;&gt;`), this function converts the object value to a string representation for form submission.<br/>If the shape of the object is `&#123; value, label &#125;`, the value will be used automatically without needing to specify this prop. |
| **isItemEqualToValue** | `(...) => boolean` | — | Custom comparison logic used to determine if a combobox item value matches the current selected value. Useful when item values are objects without matching referentially.<br/>Defaults to `Object.is` comparison. |
| **inputValue** | `string \| number \| readonly string[]` | — | The input value of the combobox. Use when controlled. |
| **defaultInputValue** | `string \| number \| readonly string[]` | — | The uncontrolled input value when initially rendered.<br/><br/>To render a controlled input, use the `inputValue` prop instead. |
| **onInputValueChange** | `(...) => void` | — | Callback fired when the input value of the combobox changes. |
| **autoComplete** | `"none" \| "list" \| "both" \| "inline"` | `'list'` | Controls how the component behaves with respect to list filtering and inline autocompletion.<br/>- `list` (default): items are dynamically filtered based on the input value. The input value does not change based on the active item.<br/>- `both`: items are dynamically filtered based on the input value, which will temporarily change based on the active item (inline autocompletion).<br/>- `inline`: items are static (not filtered), and the input value will temporarily change based on the active item (inline autocompletion).<br/>- `none`: items are static (not filtered), and the input value will not change based on the active item. |
| **formAutoComplete** | `string` | — | Provides a hint to the browser for autofill on the hidden input element.<br/>@see https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Attributes/autocomplete |
| **itemToStringLabel** | `(...) => string` | — | When the item values are objects (`&lt;Combobox.Item value=&#123;object&#125;&gt;`), this function converts the object value to a string representation for display in the input.<br/>If the shape of the object is `&#123; value, label &#125;`, the label will be used automatically without needing to specify this prop. |
| **autoHighlight** | `boolean \| "always"` | `false` | Whether the first matching item is highlighted automatically.<br/>- `false`: do not highlight automatically.<br/>- `true`: highlight after the user types and keep the highlight while the query changes.<br/>- `'always'`: highlight the first item as soon as the list opens. |
| **keepHighlight** | `boolean` | `false` | Whether the highlighted item should be preserved when the pointer leaves the list. |
| **highlightItemOnHover** | `boolean` | `true` | Whether moving the pointer over items should highlight them.<br/>Disabling this prop allows CSS `:hover` to be differentiated from the `:focus` (`data-highlighted`) state. |
| **openOnInputClick** | `boolean` | `true` | Whether the popup opens when clicking the input. |
| **required** | `boolean` | `false` | Whether the user must choose a value before submitting a form. |
| **readOnly** | `boolean` | `false` | Whether the user should be unable to choose a different option from the popup. |
| **onItemHighlighted** | `(...) => void` | — | Callback fired when an item is highlighted or unhighlighted.<br/>Receives the highlighted item value (or `undefined` if no item is highlighted) and event details with a `reason` property describing why the highlight changed.<br/>The `reason` can be:<br/>- `'keyboard'`: the highlight changed due to keyboard navigation.<br/>- `'pointer'`: the highlight changed due to pointer hovering.<br/>- `'none'`: the highlight changed programmatically. |
| **inputRef** | `Ref<HTMLInputElement>` | — | A ref to the hidden input element. |
| **filteredItems** | `readonly any[] \| readonly Group<any>[]` | — | Filtered items to display in the list.<br/>When provided, the list will use these items instead of filtering the `items` prop internally.<br/>Use when you want to control filtering logic externally with the `useFilter()` hook. |
| **virtualized** | `boolean` | `false` | Whether the items are being externally virtualized. |
| **limit** | `number` | `-1` | The maximum number of items to display in the list. |
| **locale** | `LocalesArgument` | — | The locale to use for string comparison.<br/>Defaults to the user's runtime locale. |
| **submitOnItemClick** | `boolean` | `false` | Whether clicking an item should submit the owning form. |
| **items*** | `readonly Group<any>[]` | — | — |
